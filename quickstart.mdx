## Prerequisites

Before you begin, make sure you have:

<AccordionGroup>
  <Accordion icon="github" title="GitHub Copilot subscription">
    You need an active GitHub Copilot subscription (Individual, Business, or Enterprise). If you don't have one, [sign up for GitHub Copilot](https://github.com/features/copilot).
  </Accordion>

  <Accordion icon="code" title="Supported IDE">
    Install one of the following:
    - **VS Code** (recommended) - [Download](https://code.visualstudio.com/)
    - **VS Code Insiders** - [Download](https://code.visualstudio.com/insiders/)
    - **Visual Studio** - [Download](https://visualstudio.microsoft.com/)
  </Accordion>

  <Accordion icon="docker" title="Docker (for MCP Server)">
    Docker must be installed and running to use the MCP server. [Download Docker Desktop](https://www.docker.com/products/docker-desktop/).
  </Accordion>
</AccordionGroup>

            "command": "docker",

              "-i",

  <Tab title="Manual Installation">
    If you prefer to manually configure the MCP server:

    <Steps>
      <Step title="Open MCP settings">
        In VS Code:
        1. Open Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`)
        2. Type "MCP: Edit Settings"
        3. Select and open the MCP configuration file

        In Visual Studio:
        1. Go to Tools > Options
        2. Navigate to GitHub Copilot > MCP Servers
        3. Click "Edit Configuration"
      </Step>

      <Step title="Add the configuration">
        Copy the appropriate configuration from above and paste it into your MCP settings file.
      </Step>

      <Step title="Restart your IDE">
        Restart VS Code or Visual Studio to activate the MCP server.
      </Step>

      <Step title="Verify installation">
        Open GitHub Copilot chat and verify the MCP server is running by checking for available tools.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Your First Agent

Now that the MCP server is installed, let's install and use your first custom agent!

<Steps>
  <Step title="Browse available agents">
    Visit the [Agents Catalog](/resources/agents/catalog) or use the MCP server tools in Copilot chat to search for agents.

    <Tip>
      Try asking Copilot: "Show me available agents for React development" or "Find agents for Azure infrastructure"
    </Tip>
  </Step>

  <Step title="Install an agent">
    Let's install a popular agent as an example. Choose one of these methods:

    **Option 1: One-click install (from catalog)**

    In the [Agents Catalog](/resources/agents/catalog), click the "Install in VS Code" badge next to any agent.

    **Option 2: Manual download**

    1. Navigate to the [agents directory](https://github.com/github/awesome-copilot/tree/main/agents) on GitHub
    2. Find an agent (e.g., `context7.agent.md` for up-to-date library documentation)
    3. Click "Raw" and save the file to your repository

    **Option 3: Using MCP tools**

    In Copilot chat:
    ```
    @workspace Search for agents related to "documentation"
    ```
  </Step>

  <Step title="Activate the agent">
    Once installed, the agent becomes available in your GitHub Copilot interface.

    **In VS Code:**
    - Open Copilot Chat
    - Type `@` to see available agents
    - Select your newly installed agent

    **In Coding Agent (CCA):**
    - When assigning an issue to Copilot
    - Select the custom agent from the provided list
  </Step>

  <Step title="Use the agent">
    Let's test the Context7 Expert agent (if you installed it):

    ```
    @context7-expert What are the best practices for Next.js 14 routing?
    ```

    The agent will use the Context7 MCP server to fetch up-to-date documentation and provide accurate, current information.
  </Step>
</Steps>

## Understanding Agent Resources

Many agents work with **MCP servers** to provide enhanced capabilities. Here's what you need to know:

<AccordionGroup>
  <Accordion icon="server" title="What are MCP servers?">
    MCP (Model Context Protocol) servers are services that provide tools and resources to AI agents. They enable agents to:
    - Search external documentation
    - Access APIs and databases
    - Retrieve real-time information
    - Perform specialized computations

    Each agent's documentation specifies which MCP servers it requires.
  </Accordion>

  <Accordion icon="plug" title="Installing MCP servers for agents">
    Some agents require additional MCP servers beyond the Awesome Copilot server. For example:

    **Context7 Expert** requires the Context7 MCP server:
    ```json
    {
      "context7": {
        "type": "http",
        "url": "https://mcp.context7.com/mcp",
        "headers": {
          "CONTEXT7_API_KEY": "your-api-key-here"
        }
      }
    }
    ```

    Each agent's page includes "Install MCP" badges for required servers.
  </Accordion>

  <Accordion icon="key" title="Environment variables and secrets">
    Some MCP servers require API keys or credentials. These should be:
    - Stored in your IDE's secrets manager
    - Never committed to version control
    - Referenced using `${{ secrets.SECRET_NAME }}` syntax

    Example:
    ```json
    "headers": {
      "Authorization": "Bearer ${{ secrets.APIFY_TOKEN }}"
    }
    ```
  </Accordion>
</AccordionGroup>

## Adding Instructions

Instructions automatically apply coding standards to files based on patterns. Let's add one:

<Steps>
  <Step title="Choose an instruction">
    Browse the [Instructions Catalog](/resources/instructions/catalog) or search for instructions matching your tech stack.

    Example: `typescript.instructions.md` for TypeScript best practices
  </Step>

  <Step title="Download the file">
    Save the `.instructions.md` file to your repository's `instructions/` folder (create it if it doesn't exist).
  </Step>

  <Step title="Automatic activation">
    Instructions automatically activate when you work on matching files. Check the frontmatter to see which file patterns trigger the instruction.

    Example frontmatter:
    ```yaml
    ---
    description: 'TypeScript best practices and coding standards'
    applyTo: '**.ts, **.tsx'
    ---
    ```
  </Step>
</Steps>

## Using Skills

Skills are self-contained folders with specialized capabilities:

<Steps>
  <Step title="Find a skill">
    Browse the [Skills Catalog](/resources/skills/catalog) for tasks like:
    - Terraform analysis
    - Web app testing
    - Documentation generation
    - Code migration
  </Step>

  <Step title="Install the skill">
    Skills are folders with bundled resources. Clone or download the entire skill folder to your repository.
  </Step>

  <Step title="Load the skill">
    Skills can be:
    - Loaded on-demand in Copilot chat
    - Referenced by custom agents
    - Installed as part of plugins
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-xmark" title="MCP server not connecting">
    1. Verify Docker is running: `docker ps`
    2. Check the MCP server configuration in your IDE settings
    3. Restart your IDE after configuration changes
    4. Review IDE logs for MCP-related error messages
  </Accordion>

  <Accordion icon="triangle-exclamation" title="Agent not appearing">
    1. Ensure the agent file is in the correct location
    2. Check that the frontmatter is valid YAML
    3. Restart your IDE to reload agent definitions
    4. Verify the agent file has the `.agent.md` extension
  </Accordion>

  <Accordion icon="lock" title="MCP authentication errors">
    1. Verify API keys are correctly set in secrets
    2. Check the MCP server headers configuration
    3. Ensure environment variables are accessible
    4. Review the specific MCP server's authentication documentation
  </Accordion>

  <Accordion icon="file-slash" title="Instructions not activating">
    1. Check the `applyTo` pattern matches your file extension
    2. Verify the instruction file is in the `instructions/` folder
    3. Ensure frontmatter includes both `description` and `applyTo` fields
    4. Try opening a file that matches the pattern
  </Accordion>
</AccordionGroup>

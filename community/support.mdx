## How to Get Help

This project uses GitHub issues to track bugs and feature requests. We're here to help you get the most out of the Awesome GitHub Copilot repository.

### Before Creating an Issue

<Steps>
  <Step title="Search existing issues">
    Please search the existing issues before filing new issues to avoid duplicates. Someone else may have already asked the same question or reported the same bug.
  </Step>
  <Step title="Check the documentation">
    Review the documentation to see if your question is already answered:
    - [Contributing Guide](/community/contributing)
    - [FAQ](/community/faq)
    - Resource-specific documentation
  </Step>
  <Step title="Gather information">
    If you're reporting a bug, gather relevant information such as:
    - Steps to reproduce the issue
    - Expected vs. actual behavior
    - Error messages or screenshots
    - Your environment (OS, Node.js version, etc.)
  </Step>
</Steps>

### Creating a New Issue

For help or questions about using this project, or to report bugs and request features, please [raise an issue on GitHub](https://github.com/github/awesome-copilot/issues).

<CardGroup cols={2}>
  <Card title="Bug Report" icon="bug">
    Found a bug? Create an issue describing:
    - What you expected to happen
    - What actually happened
    - Steps to reproduce
  </Card>
  <Card title="Feature Request" icon="lightbulb">
    Have an idea? Create an issue explaining:
    - The problem you're trying to solve
    - Your proposed solution
    - Why this would be useful to others
  </Card>
  <Card title="Question" icon="question">
    Need help? Create an issue asking:
    - What you're trying to accomplish
    - What you've already tried
    - Where you're stuck
  </Card>
  <Card title="Documentation" icon="book">
    Found unclear docs? Create an issue noting:
    - Which documentation page
    - What's confusing or missing
    - Suggestions for improvement
  </Card>
</CardGroup>

## Support Policy

**Awesome GitHub Copilot** is under active development and maintained by GitHub and Microsoft staff **and the community**. We will do our best to respond to support, feature requests, and community questions in a timely manner.

<Note>
Support for this project is limited to the resources listed on this page. GitHub's standard support channels do not cover this open source project.
</Note>

### What to Include in Your Issue

To help us assist you effectively, please include:

- **Clear title**: Summarize your issue in a few words
- **Detailed description**: Explain the problem or question thoroughly
- **Context**: Include relevant information about your setup and what you're trying to do
- **Code examples**: If applicable, share code snippets or configuration files
- **Error messages**: Include the full text of any error messages
- **Screenshots**: Visual aids can be very helpful

### Response Times

As a community-driven project:

- Maintainers and community members respond as time permits
- Complex issues may take longer to resolve
- We prioritize issues that affect multiple users
- Security issues are handled according to our [Security Policy](/community/security)

## GitHub Support Policy

</Warning>

- [GitHub Copilot Documentation](https://docs.github.com/en/copilot)

## Getting Involved

The best way to get help is often to help others! Consider:

- Answering questions in GitHub issues
- Reviewing pull requests
- Improving documentation
- Sharing your own instructions, agents, or skills
- Testing new contributions

<Note>
Every contribution, no matter how small, helps make this resource better for the entire GitHub Copilot community.
</Note>

## Contact

For questions or help, please raise an issue on GitHub. The community and maintainers will do their best to assist you.

For security issues, contact **opensource-security@github.com** (see [Security Policy](/community/security) for details).

For Code of Conduct violations, contact **opensource@github.com** (see [Code of Conduct](/community/code-of-conduct) for details).
